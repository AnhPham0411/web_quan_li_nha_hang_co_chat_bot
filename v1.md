### 1.1 Khôi Phục Mật Khẩu

**Vấn đề:** Trang `forgot-password` chỉ là UI placeholder, không gửi email thực sự.

**Giải pháp:** Dùng [Resend](https://resend.com) (serverless-native, miễn phí 3000 email/tháng).

**Schema mới — thêm vào `prisma/schema.prisma`:**
```prisma
model PasswordResetToken {
  id        String   @id @default(uuid())
  userId    String
  token     String   @unique
  expiresAt DateTime
  used      Boolean  @default(false)
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

**API mới:**

| Route | Method | Mô tả |
|-------|--------|-------|
| `/api/auth/forgot-password` | POST | Tạo token, gửi email link reset |
| `/api/auth/reset-password` | POST | Verify token, hash mật khẩu mới |

**Luồng:**
1. User nhập email → `POST /api/auth/forgot-password`
2. Server tạo UUID token, lưu DB với `expiresAt = now + 1h`
3. Gửi email có link: `https://domain.com/reset-password?token=<uuid>`
4. User click link → trang `/reset-password/[token]`
5. User nhập mật khẩu mới → `POST /api/auth/reset-password`
6. Server verify token chưa hết hạn, chưa dùng → bcrypt hash → lưu DB → đánh dấu `used = true`

**Files cần tạo/sửa:**
- `src/app/(auth)/forgot-password/page.tsx` — cập nhật form gọi API thật
- `src/app/(auth)/reset-password/[token]/page.tsx` — trang mới
- `src/app/api/auth/forgot-password/route.ts` — tạo mới
- `src/app/api/auth/reset-password/route.ts` — tạo mới
- `prisma/schema.prisma` — thêm model
- `src/lib/email.ts` — Resend client wrapper

### 4.1 Order Tại Bàn Qua QR Code

**Mô tả:** Khi khách đang ngồi (`table.status = SERVING`), quét QR → order thêm không cần gọi nhân viên.

**Schema mới:**
```prisma
model Order {
  id        String      @id @default(uuid())
  tableId   String
  status    OrderStatus @default(PENDING)
  notes     String?     @db.Text
  createdAt DateTime    @default(now())
  updatedAt DateTime    @updatedAt

  table Table       @relation(fields: [tableId], references: [id])
  items OrderItem[]
}

model OrderItem {
  id         String   @id @default(uuid())
  orderId    String
  menuItemId String
  quantity   Int
  notes      String?

  order    Order    @relation(fields: [orderId], references: [id], onDelete: Cascade)
  menuItem MenuItem @relation(fields: [menuItemId], references: [id])
}

enum OrderStatus {
  PENDING
  CONFIRMED
  PREPARING
  SERVED
  CANCELLED
}
```

**Pages mới:**
- `/order/[tableId]` — Trang public (không cần login), hiển thị menu + giỏ hàng
- `/admin/orders/page.tsx` — Admin xem orders theo bàn/thời gian

**Luồng:**
1. Admin in QR code cho từng bàn (`/order/[tableId]`)
2. Khách quét QR → vào trang menu
3. Khách chọn món → gửi order
4. API `POST /api/orders` → tạo Order + OrderItems, giảm `stockQuantity`
5. Pusher notify admin: "Bàn 3 gọi thêm: 2 Bò Wagyu, 1 Coca"
6. Admin confirm → `PATCH /api/orders/[id]` → status = CONFIRMED

### 5.1 Rate Limiting

**Package:** `@upstash/ratelimit` + `@upstash/redis` (serverless-native, miễn phí tier)

**Áp dụng cho:**

| API Route | Giới hạn |
|-----------|---------|
| `POST /api/reservations` | 5 req/phút per IP |
| `POST /api/auth/register` | 3 req/phút per IP |
| `POST /api/auth/forgot-password` | 2 req/phút per IP |
| `POST /api/webhook/dialogflow` | 30 req/phút (token đã verify) |
| `GET /api/menu` | 60 req/phút per IP |

**Implementation trong middleware hoặc từng route:**
```typescript
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(5, "1 m"),
});
```

- `GET /api/menu` — ít thay đổi, truy vấn nhiều → cache 5 phút
- `GET /api/tables` — thay đổi thường → cache 30 giây, invalidate khi PATCH

**Implementation:**
```typescript
// src/lib/cache.ts
import { Redis } from "@upstash/redis";
const redis = Redis.fromEnv();

export async function getCached<T>(key: string, fetcher: () => Promise<T>, ttl = 300): Promise<T> {
  const cached = await redis.get<T>(key);
  if (cached) return cached;
  const data = await fetcher();
  await redis.setex(key, ttl, JSON.stringify(data));
  return data;
}
```

## 8. Database Schema Mở Rộng

```prisma
// Thêm vào prisma/schema.prisma

model PasswordResetToken {
  id        String   @id @default(uuid())
  userId    String
  token     String   @unique
  expiresAt DateTime
  used      Boolean  @default(false)
  createdAt DateTime @default(now())
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model Order {
  id        String      @id @default(uuid())
  tableId   String
  status    OrderStatus @default(PENDING)
  notes     String?     @db.Text
  createdAt DateTime    @default(now())
  updatedAt DateTime    @updatedAt
  table     Table       @relation(fields: [tableId], references: [id])
  items     OrderItem[]
}

model OrderItem {
  id         String   @id @default(uuid())
  orderId    String
  menuItemId String
  quantity   Int
  notes      String?
  order      Order    @relation(fields: [orderId], references: [id], onDelete: Cascade)
  menuItem   MenuItem @relation(fields: [menuItemId], references: [id])
}

model Shift {
  id        String   @id @default(uuid())
  staffId   String
  date      DateTime @db.Date
  startTime DateTime
  endTime   DateTime
  notes     String?
  createdAt DateTime @default(now())
  staff     User     @relation(fields: [staffId], references: [id])
}

model PushSubscription {
  id           String   @id @default(uuid())
  userId       String?
  endpoint     String   @unique @db.Text
  p256dhKey    String
  authKey      String
  createdAt    DateTime @default(now())
  user         User?    @relation(fields: [userId], references: [id])
}

enum OrderStatus {
  PENDING
  CONFIRMED
  PREPARING
  SERVED
  CANCELLED
}
```
